## But

Faire que les noms de domaine OVH `domaine.fr` et `*.domaine.fr` ainsi que la génération des certificats SSL Let's Encrypt fonctionnent au travers du reverse proxy  Nginx Proxy Manager.

Gérer avec OVH la mise à jour automatique de l'IP dynamique de la box Internet _(DynDns)_.

## Prérequis OVH et Proxmox

### OVH

Se connecter sur `https://www.ovhcloud.com/fr/`

&rarr; Tableau de bord &rarr; Noms de domaine  
&rarr; Cliquer sur le lien `domaine.fr`

Le domaine `domaine.fr` doit être enregistré.  
Les serveurs DNS d'OVH doivent être actifs.  
La sécurité DNSSEC doit être désactivée _(cause = IP dynamique)._  
L'option DNS Anycast doit être activée _(couverture DNS mondiale)_.

### Proxmox

Une IP dynamique  _(dû à l'opérateur Internet)_.  
Un Nginx Proxy Manager installé et fonctionnel.

## DynDns côté OVH

&rarr; Tableau de bord &rarr; Zones DNS  
&rarr; Cliquer sur le lien `domaine.fr`

<!-- more -->

Observer les entrées telles que :  
A `domaine.fr.` &rarr; Cible = IP box Internet  
A `www.domaine.fr.` &rarr; Cible = IP box Internet  
CNAME `ftp.domaine.fr.` &rarr; Cible = domaine.fr.

### Gestion du DynHost

Étape A – Créer l’utilisateur DynHost :  
&rarr; Tableau de bort &rarr; Noms de domaine  
&rarr; Clic sur `domaine.fr` &rarr; Onglet DynHost

Bouton Ajouter un DynHost :  
Sous-domaine = proxy _(pour `proxy.domaine.fr.`)_  
IP = IP externe de la box Internet ou 0.0.0.0  
Bouton Valider

La mise à jour automatique de l'IP se fera ultérieurement.

Bouton Gérer les accès :  
&rarr; Bouton Créer un identifiant :  
Suffixe de l'identifiant = domaine.fr - user  
Sous-domaine = proxy  
Mot de passe = 12 caractères _(pas de caractères spéciaux)_  
Confirmation =  
Bouton Valider

Cela crée implicitement la liaison :  
`proxy.domaine.fr.` &rarr; IP dynamique de la box Internet.

### Enregistrements A et CNAME

Modifier les A et CNAME par défaut comme suit :  
&rarr; Tableau de bort &rarr; Zones DNS  
&rarr; Clic sur `domaine.fr`

Enregistrement A :  
Garder l'enregistrement `domaine.fr.`  
avec Cible = IP box Internet  
Créer un enregistrement pour le sous-domaine `proxy.domaine.fr.`  
avec Cible = IP box Internet  
Supprimer `www.domaine.fr.` _(optionnel)_  

Enregistrement CNAME :  
Remplacer `ftp.domaine.fr.` par `*.domaine.fr.`  
avec Cible = `proxy.domaine.fr.`  

## DynDnS côté Proxmox

### Client Linux ddclient

Installer le client Linux _ddclient_ dans le dossier /root :

```bash
apt install ddclient
```

Une fenêtre de configuration s'ouvre :  
\- Choisir Autre  
\- Protocole =  dyndns2  
\- Serveur de DNS dynamique =  `www.ovh.com`  
\- Identifiant = `domaine.fr-user` _(user déclaré dans DynHost d'OVH)_  
\- MDP = _(mdp déclaré dans DynHost d'OVH)_  
\- Confirmation MDP =  
\- Méthode de découverte d'adresse IP = Service ... basée sur le web  
\- Hôtes à mettre à jour = `proxy.domaine.fr`  

Les données entrées sont inscrites dans le fichier /etc/ddclient.conf qui doit ressembler exactement à ceci :

```markdown
# Configuration file for ddclient generated by debconf
#
# /etc/ddclient.conf

protocol=dyndns2
use=web, web=checkip.dyndns.com
server=www.ovh.com
login=domaine.fr-user
password=mot-de-passe
proxy.domaine.fr
```

Vérifier le résultat du service ddclient:  

```bash
systemctl status ddclient
```

ou  

```bash
ddclient -verbose -file /etc/ddclient.conf
```

ou

```bash
curl `https://www.ovh.com/nic/update?system=dyndns&hostname=proxy.domaine.fr&myip=xx.xx.xx.xx" --user "domaine.fr-user:mot-de-passe`
```

### Tâche Crontab

Créer une tâche _cron_ pour une mise à jour automatique de l'IP dynamique toutes les 5 minutes :

```bash
crontab -e
```

Ajouter en fin de fichier la ligne suivante :

```markdown
*/5 * * * * /usr/bin/ddclient -quiet
```

## Nginx Proxy Manager

### Certificats Let's Encrypt

Au préalable, se connecter sur OVH puis ouvrir l'URL :  
`https://api.ovh.com/createToken/index.cgi`

Ceci ouvre la page Create API Keys.

Remplir les champs de la page comme suit :  
Application name = DNS_Certbot  
Application description = Renouvellement automatique certificats SSL  
Validity = UNlimited

GET  /domain/zone/\* &rarr; +  
POST /domain/zone/\* &rarr; +  
PUT  /domain/zone/\* &rarr; +  
DELETE /domain/zone/\* &rarr; Bouton Create

Noter le contenu du retour :  
Application Key  
Application Secret  
Consumer Key

Créer ensuite le certificat SSL depuis Nginx Proxy Manager.

&rarr; SSL Certificates &rarr; Add Certificate :  
Type = Let’s Encrypt via DNS  
Domain Names = `domaine.fr` et `*.domaine.fr`  
DNS Provider = Sélectionner OVH

Credentials File Content, renseigner :  
dns_ovh_endpoint = ovh-eu _(pour l'europe)_  
Application Key  
Application Secret  
Consumer Key

Propagation Seconds = 120 ou 180  
&rarr; Bouton Save

### Proxy Hosts

Configurer les Proxy Hosts.

Pour chaque domaine ou sous-domaine, Ex = xx.domaine.fr :  
-- Onglet Details --  
Domain Names = xx.domaine.fr  
Scheme = http  
Forward Hostname / IP = IP interne VM ou conteneur  
Forward Port = Numéro de port

-- Onglet SSL --  
SSL Certificate = Wildcard pour xx.domaine.fr  
Force SSL = Oui  
HTTP/2 Support = Oui  
HSTS Enabled = Oui

## Vérifications finales

Tests DNS avec la Cde _dig_ :

```bash
dig proxy.domaine.fr
dig xx.domaine.fr
```

Les deux réponses doivent retourner la même IP publique.

Tests HTTPS :  
`https://xx.domaine.fr`  
`https://yy.domaine.fr`  
`https://zz.domaine.fr`

Les pages Web concernées doivent s'afficher normalement.
